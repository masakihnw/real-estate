# UI/UX 大幅刷新 提案書

作成日: 2026-06-11
対象: real-estate-ios（SwiftUI / 約100ファイル・33,000行）

## 0. 前提とゴール

本提案は以下のゴールに向けた、現状調査（コードベース全数調査）と刷新案・追加機能提案である。

| ゴール | 本提案での解釈 |
|---|---|
| 必要なタイミングで必要な情報 | 利用シーンを「①朝晩の新着・値下げチェック」「②暇つぶし探索」「③腰を据えた検討」「④現地内見」の4モードに分解し、各モードで最短経路を設計する |
| 毎日触りたくなるデザインと体験 | 「今日の動き」を主役にしたホーム刷新＋スワイプ探索の強化。モーション・ハプティクスで操作の気持ちよさを作る |
| 毎日触ってもストレスにならない快適性 | スクロール量・タップ数・待ち時間の削減を数値目標で管理（後述のKPI） |
| 開発途中の画面がない | 開発者向け機能の隔離、デッドコードの削除、データ欠損時の見せ方の統一 |
| 情報のメリハリ | 各画面で「最重要の1情報」を定義し、それ以外を段階的開示（progressive disclosure）に落とす |

**確認済みの前提**: 本アプリは自分（家族）専用ツール。汎用プロダクト向けのオンボーディング強化は不要で、むしろ「個人前提だから許される思い切った最適化」（buyer_profile 直結、通勤先固定、判断基準の単一化）を強みとして伸ばす。デザイン方向はハイブリッド（発見＝写真主役、検討＝データ主役）。

## 1. 現状調査サマリー

### 1.1 アプリ構造（情報アーキテクチャ）

現在は6タブ構成（`ContentView.swift`）:

| タブ | 内容 | 日次利用での役割 |
|---|---|---|
| ダッシュボード | 統計・スワイプ導線・タイムライン等 9セクション | 毎日の起点（だが情報過多） |
| 物件 | 中古一覧＋フィルタ＋比較 | 検討の中心 |
| 地図 | 物件ピン＋ハザード10レイヤー | 補助的 |
| マイリスト | お気に入り一覧（物件タブと同一View） | 検討の中心 |
| 成約 | 成約事例の一覧/地図 | 参照頻度低（検討時のみ） |
| 設定 | 通知・データ・My指標・開発者ツール・アカウント | 稀 |

主要な構造的課題:

- **ダッシュボードが9セクション・2〜3画面分**（`DashboardView.swift` 1,014行）。「今日の変化を30秒で確認」したいのに、統計カード→スワイプ→クイックフィルタ→AI→スコア分布→タイムライン→ウォッチ値下げ→価格変動→エリアランキングと一列に並び、毎日見るべき情報（新着・値下げ）とたまに見れば良い情報（スコア分布・エリアランキング）が同じ重みで混在。
- **「物件」と「マイリスト」が同一Viewの引数違い**で、タブとしての役割分担が曖昧。
- **「成約」が独立タブ**だが、成約事例は単体で眺めるものではなく「この物件は相場と比べてどうか」の文脈で必要になる情報。独立タブはタブ一等地の無駄遣い。
- **未接続のデッドコード**: `NopedListingsView.swift` はどこからも到達できない（Nopeの解除手段がUI上存在しない）。
- **開発者向け機能がユーザー動線に混在**: 設定 > 詳細設定に「スクレイピング条件」「スクレイピングログ」「Supabase APIトグル」「カスタムURL」、このアプリについてに「診断情報」。ゴール「必要なものだけが表示される」に反する。

### 1.2 主要画面のUX課題（定量）

| 画面 | 計測値 | 課題 |
|---|---|---|
| 物件詳細 `ListingDetailView.swift` | **3,133行・25セクション・スクロール5〜8画面分** | 1物件の判断に必要な情報が縦一列。お金ツール5種はアイコンのみのボタン列で中腹に埋没。内見メモは最下部 |
| 一覧 `ListingListView.swift` | 1,975行 | 行there情報は適切だが、フィルタ・ソートが重い（下記） |
| フィルタ `ListingFilterSheet.swift` | **11セクション・数値フィルタ16種以上** | どのフィールドにデータが入っているか分からないまま絞ると0件になる。条件の保存（保存検索）がない |
| ソート | **48択のフラットなメニュー** | 「沖式儲かる確率」と「値上がり率予測」の違いが分からないまま選ぶ。グルーピングなし |
| 地図 `MapTabView.swift` | 1,830行・ハザード10レイヤー | レイヤー選択が説明文依存。ピンのコールアウトが最小限で「この物件、浸水域？」に答えられない |
| スワイプ | カード設計は良好 | 中断すると進捗が消える。ハプティクスなし。Skipの意味が不明瞭 |
| 比較 `ComparisonView.swift` | 横スクロールグリッド | iPhoneで2物件比較ですら横スクロール必須 |

### 1.3 ビジュアルデザインの現状（デザインシステム監査）

`Design/DesignSystem.swift` にセマンティックカラー（スコアS〜D、AI accent、価格上下、ソース別色等）は定義済みだが、**採用率が極端に低い**:

| 項目 | 実態 |
|---|---|
| 色 | ハードコード色 **394箇所**（45ファイル）。不透明度は 0.04〜0.9 の12段階がアドホックに散在。トークン採用率 5% 未満 |
| タイポグラフィ | フォント指定 **832箇所**のうちデザイントークン（`ListingObjectStyle`）使用は **11箇所（1.3%）**。`.system(size: 8〜13)` のハードコードが37箇所以上（Dynamic Type非対応） |
| 角丸 | トークンは12だが、実際は `cornerRadius: 8` が49箇所で最多。3/4/5/6/8/10/12/14/20 が混在 |
| 余白 | トークン4つのみ。`.padding(14)` が23箇所など20種以上のマジックナンバー |
| 影 | トークンなし。6種類の組み合わせが各Viewで独自定義 |
| ハプティクス | **0箇所**（スワイプのLike/Nopeにすら無い） |
| ダークモード | セマンティックカラー経由で概ね対応だが、LoginViewの青グラデ等ハードコード箇所は未検証。Assets.xcassetsにカスタム色なし（AccentColor空） |
| コンポーネント | ScoreBadge/HighlightBadgeView等は共通化済みだが、カード・行・バッジ・価格フォーマットは画面ごとに再実装（DashboardだけでStatCard/PriceMoverRow/TimelineFeedRow/WatchlistPriceDropRowが別実装） |
| アクセシビリティ | 67箇所（11ファイル）に偏在。チャート・色のみのバッジは読み上げ不可 |

**結論**: 「デザインシステムはあるが、機能していない」。刷新の第一歩はビジュアルの方向性以前に、トークンの拡充と強制（lint）。

### 1.4 データ資産の棚卸し（提案の材料）

このアプリの最大の強みは**個人向けとは思えないデータの厚み**。AIスコアリング（5軸）、AI推奨（1〜5点＋フラグ＋ネクストアクション）、住まいサーフィン（沖式・ランキング・5/10年シミュレーション）、成約事例（reinfolib）、ハザード（GSI 10種）、通勤時間（2社分）、人口動態（e-Stat）、価格履歴、重複検知。

一方で**取得済みなのにUIに出ていない/活かせていないデータ**が多数ある:

| データ | 現状 | 活用余地 |
|---|---|---|
| マンションレビュー（管理品質） | 取得済み・**非表示** | 詳細の「建物」評価、比較表の行 |
| 周辺施設スコア（Google Places: 学校・店・公園） | 取得可能・UIなし | 家族向けに価値大 |
| 用途地域 `zoning` | フィールド有・**非表示** | 将来の眺望・環境リスク判断 |
| 長期修繕情報（AI抽出） | 抽出済み・非表示 | 資金計画 |
| 同建物の住戸別価格判定 `ssPriceJudgments` | 詳細にリスト表示のみ | 「この建物ならどの住戸が買いか」 |
| ローン残高予測 `ssLoanBalance5yr/10yr` | 表示のみ | 売却シミュレーション（残債割れ判定）に組込み |
| feature_tags | リスト表示のみ | フィルタ・推薦に未使用 |
| 賃料相場・利回り | **未取得** | 出口戦略の説得力向上（新規データ） |

## 2. 刷新コンセプト

### 2.1 4つの利用モードと「必要なタイミングで必要な情報」

| モード | 頻度 | 滞在時間 | 主役の情報 | デザイン |
|---|---|---|---|---|
| ① 今日のチェック | 毎日 朝晩 | 30秒〜2分 | 新着・値下げ・ウォッチ物件の変化 | **変化диフ主役**。変わったものだけ見せる |
| ② 暇つぶし探索 | 毎日 隙間時間 | 2〜10分 | 物件写真・直感的な好み | **写真主役**・全画面・スワイプ |
| ③ 腰を据えた検討 | 週末 | 10分〜 | AI分析・相場・シミュレーション | **データ主役**・比較・密度高 |
| ④ 現地内見 | 不定期 | 現地で | チェックリスト・メモ・周辺相場 | 片手操作・オフライン耐性 |

現状のアプリは③に全振りした作りで、①②が③の画面を間借りしている。刷新の核は **①②を独立した一級体験に昇格させること**。

### 2.2 デザイン原則（5箇条）

1. **One screen, one hero** — 各画面で「一番伝える情報」を1つ決め、面積・タイポ・色のコントラストをそこに集中。それ以外は段階的開示。
2. **変化を祝う** — 毎日触る理由は「変化」。新着・値下げ・順位変動を、色とモーションで最初に目に入れる。変化がない日は「変化なし」を1行で言い切り、即②に誘導する。
3. **写真で惚れさせ、データで決めさせる** — 発見系画面（Today・スワイプ・一覧カード）は写真とAIバッジ。検討系画面（詳細・比較）は数値とチャート。同じ物件オブジェクトでも画面のモードで表現を切り替える（OOUIの徹底）。
4. **判断の言語はS〜Dグレード** — AIスコア・沖式・価格妥当性など多数の指標を、ユーザーが最初に見る場所では「グレード＋一言」に集約。内訳はタップで開く。
5. **触感のあるUI** — 全主要操作にハプティクス、状態変化にスプリングモーション。iOS 26 Liquid Glassを全面採用（フォールバック維持）。

### 2.3 快適性のKPI（刷新の合格基準）

- 朝のチェック完了（新着＋値下げ＋ウォッチ確認）: **アプリ起動から30秒以内・スクロール1画面以内**
- 一覧→詳細で「買い/見送りの一次判断」: **詳細1画面目（スクロールなし）で完結**
- お金シミュレーションへの到達: **詳細から1タップ**
- ソート変更: **2タップ・選択肢は1階層8択以内**（残りはグループ内）
- スワイプ: 中断→再開で**進捗保持**

## 3. 情報アーキテクチャ刷新案

### 3.1 タブ再編: 6タブ → 4タブ

```
現在:  ダッシュボード | 物件 | 地図 | マイリスト | 成約 | 設定
提案:  今日 (Today)   | さがす (Browse) | マイリスト (Library) | 設定
```

| 新タブ | 統合される既存機能 | 役割 |
|---|---|---|
| **今日** | ダッシュボード（再設計）＋スワイプ導線 | モード①②の起点。デイリーダイジェスト |
| **さがす** | 物件一覧＋地図（リスト⇔マップをセグメントで切替）＋フィルタ | モード③の入口。成約事例は地図レイヤー＆詳細セクションに降格 |
| **マイリスト** | お気に入り＋比較＋最近見た＋Nope管理＋内見予定 | 自分の検討状況のホーム。WatchlistViewもここに統合 |
| **設定** | 通知・My指標・通勤先・買付準備。開発者機能は隔離（後述） |  |

理由:
- 「物件」と「マイリスト」は同一View。タブを分けるより、検討ステータス（未評価/気になる/見送り）で分ける方がメンタルモデルに合う。
- 「成約」タブは利用文脈がない。成約データは (a) さがす地図の「成約価格レイヤー」（ハザードと同列のレイヤーとして）、(b) 物件詳細の相場セクション、(c) 新設の「相場レポート」（Todayの週次カード）に分散吸収する。
- 地図はリストと同じ「さがす」行為の表示形式違い。Zillow/SUUMO同様、リスト⇔マップのトグルが自然。フィルタ状態も共有でき、現状の「リストとマップでフィルタが別管理」問題も解消する。

### 3.2 「今日」タブ（新ホーム）の設計

毎日のチェックを最短化する「朝刊」型。上から:

1. **ヘッダー: 今日のひとこと（AIデイリーブリーフ）** — 「新着3件。うち1件は予算内・S評価。広尾の気になる物件が▲200万円」を1〜2文で。変化ゼロの日は「今日は動きなし。新着スワイプが5件待っています」。
2. **変化カード（横スクロール、最大5枚）** — 新着/値下げ/再掲載/ウォッチ変動を1枚=1物件の写真カードで。タップで詳細、長押しでクイックアクション（♥/Nope/共有）。
3. **スワイプ入口** — 未評価が残っていれば大きめカードで「未評価12件をスワイプ」。
4. **今週の相場（週1更新の折りたたみカード）** — エリアランキング・スコア分布・成約トレンドはここに集約（毎日見せない）。
5. それ以外（タイムライン全文等）は「すべての動きを見る」リンクの先へ。

現在9セクションが常時並ぶ構成を、「毎日変わるもの3つ＋週次1つ」に圧縮する。

### 3.3 物件詳細の再設計（最重要）

25セクション縦一列 → **「1画面目で判断、タブで深掘り」** に再構成。

**1画面目（ヒーローカード）** — スクロールせずに一次判断が完結する構成:
- 写真ギャラリー（ページング、カテゴリチップ: 外観/室内/間取り）
- 物件名・価格（変動があれば旧価格打ち消し＋▲X%バッジ）
- **AI判定ストリップ**: グレードバッジ（S〜D）＋推奨度（★1〜5）＋一言サマリー＋フラグチップ（"駅近◎ 築浅 手狭リスク"）
- KPIストリップ: 月額支払い / 駅徒歩 / 広さ / 通勤時間（2社）
- 主アクション: ♥保存・共有・元サイト

**2画面目以降: 5つのセクションタブ（ピン留めセグメント）**

| タブ | 統合する既存セクション |
|---|---|
| 概要 | 物件スペック・AI抽出特徴・別ソース価格・重複候補 |
| お金 | 月額シミュレーション＋**統合シミュレーター**（現5ツールを1画面に。後述） |
| 資産性 | 沖式・レーダー・5/10年シミュレーション・成約事例・市場比較・同建物住戸判定・人口動態 |
| 環境 | 通勤・ハザード・周辺施設（新規）・用途地域（新規表示）・マンションレビュー（新規表示） |
| メモ | コメント・内見チェックリスト・内見写真・AI相談（Markdownコピー） |

- セクションタブは `safeAreaInset` で上部固定し、現在地を常に表示（既存のセクションナビを置換）。
- **お金ツール統合**: 月額/諸費用/減税/銀行比較/賃貸vs購入/リノベの6ツールを「前提条件（価格・金利・期間・頭金）を1箇所で共有する単一シミュレーター」に統合。タブ切替で結果ビューだけ替わる。現状の「ツール間で金利を二重入力」「アイコンだけのボタン6連発」を解消。
- データ未取得のセクションは「見出しだけ薄く出す」のをやめ、**非表示＋概要タブ末尾に「未取得: 沖式・レビュー」と1行表記**（開発途中感の排除）。

### 3.4 さがす（一覧・フィルタ・地図）

- **ソート48択 → 8つの代表ソート＋「詳細ソート」**。代表: 新着順 / 価格 / 価格の割安度 / 総合スコア / AI推奨 / 駅近 / 広さ / My指標。詳細ソートはグループ化（基本・立地・お金・資産性・AI）したピッカーへ。
- **保存フィルタ（検索条件）を一級市民に**。検索バー下にチップで「予算内S評価」「広尾エリア」など保存条件を並べ、1タップ適用。新着がその条件にヒットしたらチップにバッジ。FilterStoreは既存なので拡張で対応可能。
- **数値フィルタにデータ充足率を表示**（「沖式儲かる確率 (データあり 62%)」）し、0件遭遇を予防。
- **駅ピッカーは検索フィールド先行**（3階層ドリルダウンは残すが補助に）。
- **地図**: ピンのコールアウトに「グレード・価格・徒歩・ハザードアイコン」を追加し、詳細を開かずに一次判断可能に。ハザードレイヤーは「よく使う2つ（洪水・地震リスク）」をワンタップトグル化し、残りはシートへ。成約価格をヒートマップレイヤーとして追加。

### 3.5 スワイプ（暇つぶし探索の主役化）

- **進捗の永続化**: セッション途中離脱でデッキ位置を保存、次回続きから。
- **ハプティクス＋スタンプ**: Like/Nopeで `sensoryFeedback` ＋ボタン操作時もスタンプアニメーション表示。
- **デッキの拡張**: 現在は「画像つき新着」のみ。条件付きデッキ（「S/Aだけ」「予算内だけ」「保存フィルタ◯◯」）を選べるように。新着が尽きても暇つぶしが成立する（過去の未評価物件を再配給）。
- **カードからの深掘り**: 上スワイプ or タップで「ミニ詳細」（ヒーローカード相当）をその場に展開し、戻ってスワイプ継続。
- **Skipの再定義**: 「あとで」（次回デッキの先頭に再登場）と明示。
- Nope管理（未接続の `NopedListingsView`）をマイリストの「見送り」タブとして接続し、一括解除を追加。

### 3.6 マイリスト（検討のホーム）

- セグメント: **気になる（♥） / 比較中 / 見送り / 最近見た**。
- 気になる一覧は「価格変動があったものを最上部に」自動ソート（WatchlistViewの値下げセクションを統合）。
- 比較はカード2枚並べ（縦持ちで2物件、項目は行スワイプで送る）に再設計し、横スクロールグリッドを廃止。差分が大きい行を自動ハイライト（既存 `ComparisonHighlight` を活用）。
- 内見予定の物件には「内見モード」バッジ（→ 5.6）。

### 3.7 開発者機能の隔離と「必要なものだけ」

- スクレイピング条件 / スクレイピングログ / Supabaseトグル / カスタムURL / 診断情報を、設定末尾の「**開発者**」セクション（バージョン番号7回タップで出現、UserDefaultsで永続）に移動。普段の設定画面から消す。
- `NopedListingsView` は 3.5 の通り正式接続（デッドコード解消）。
- 設定の「買付準備」「通勤先設定」は詳細設定から昇格させ、買付準備はマイリストからも到達可能に。
- ウォークスルー（7ページ）は自分専用アプリでは過剰。設定のヘルプに残すのみとし、初回自動表示は廃止。

## 4. デザインシステム刷新

### 4.1 トークン体系の拡充（DesignSystem.swift v2）

```swift
enum DS {
    enum Spacing { // 4pt グリッド
        static let xs: CGFloat = 4; static let sm: CGFloat = 8
        static let md: CGFloat = 12; static let lg: CGFloat = 16
        static let xl: CGFloat = 24; static let xxl: CGFloat = 32
    }
    enum Radius {
        static let chip: CGFloat = 8      // バッジ・チップ
        static let card: CGFloat = 16     // カード（現12から拡大、Liquid Glass映え）
        static let sheet: CGFloat = 24    // モーダル上端
    }
    enum Opacity {
        static let tintBg = 0.08; static let border = 0.16
        static let disabled = 0.4; static let overlay = 0.6
    }
    enum Shadow { // 2種のみに統一
        static let card = ShadowStyle(opacity: 0.08, radius: 8, y: 4)
        static let floating = ShadowStyle(opacity: 0.16, radius: 16, y: 8)
    }
    enum Typography { // Dynamic Type 対応の役割ベース
        static let hero = Font.largeTitle.bold()          // 価格など主役数値
        static let sectionTitle = Font.headline
        static let body = Font.subheadline
        static let label = Font.caption
        static let badge = Font.caption2.weight(.semibold)
        // .system(size:) の新規使用は禁止
    }

```

- 既存のセマンティック色（スコア・価格上下・AI・ソース・ハザード）は**Assets.xcassetsのカラーセットに移管**し、ダーク用バリアントを定義（現在カタログは空）。
- アクセントカラーを定義し（提案: 信頼感のある deep teal 系 ＋ AI 用 indigo は維持）、ブランドの軸を作る。
- **段階導入の強制**: `swiftlint` カスタムルール（または CI grep）で `Color(red:` / `.system(size:` / `cornerRadius: [0-9]` の新規追加を警告。既存394箇所は画面刷新と同時に置換。

### 4.2 コンポーネントライブラリ

最低限そろえる共通部品（既存の良い部品は昇格）:

### 4.3 モーション＆ハプティクス指針

- **ハプティクス（現状ゼロ → 標準装備）**: Like/Nope = `.impact(.medium)`、保存・適用 = `.success`、値下げ発見カード表示 = `.soft`。`sensoryFeedback` modifier（iOS 17+）で統一実装。
- **モーション**: 標準は `.spring(response: 0.35, dampingFraction: 0.8)` に統一（現在 0.2/0.25/0.3/0.35 が混在）。カード出現はスタッガード（0.05s刻み）、価格変動数値はカウントアップ。`reduceMotion` 対応を全アニメーションに義務化（現在1箇所のみ）。
- **画面遷移**: 一覧→詳細は写真の `matchedGeometryEffect`（NavigationTransition `.zoom` iOS 18+）で「写真がそのまま開く」体験に。

### 4.4 アクセシビリティ最低ライン

- チャート（レーダー・シミュレーション）に `accessibilityLabel` で要約文（「総合評価は6軸中4軸でエリア平均超え」）。
- 色のみで意味を伝える箇所（グレード・ハザード・価格上下）に記号/テキストを併記。
- `.lineLimit(1)` 箇所の Dynamic Type 検証。ハードコードサイズ全廃で大部分は自動解決。

## 5. 追加機能の提案

データ資産調査（§1.4）に基づく。**毎日触る理由を作る機能**を上位に。

### P1（刷新と同時に入れたい）

1. **AIデイリーブリーフ**（§3.2）— 既存のAI enrichment パイプラインに「日次サマリー生成」を1モジュール追加（`config/prompts/daily_brief.md`）。新着・価格変動・ウォッチ変化を1〜2文に要約してSupabaseに置き、Todayヘッダーとプッシュ通知・ウィジェットで共用。**通知が「新着3件」から「予算内のS評価が広尾に出ました」に変わる**ことが毎日開く動機になる。
2. **保存フィルタ＋条件マッチ通知**（§3.4）— 「この条件の新着だけ知りたい」を実現。既存FilterStore＋通知基盤の拡張。
3. **スワイプ進捗永続化＋条件付きデッキ**（§3.5）。
4. **ウィジェット刷新** — 現在の件数表示から「今日の1枚」（変化カード先頭の物件写真＋価格変動）へ。スモール=1物件、ミディアム=ブリーフ1文＋2物件。タップで該当詳細へディープリンク（基盤あり）。
5. **未活用データの表示**（実装コスト極小）— マンションレビュー・用途地域・長期修繕・周辺施設スコアを詳細「環境」「資産性」タブに表示。

### P2（次フェーズ）

6. **内見モード** — マイリストの物件を「内見予定」にすると、当日は現地向けレイアウト（チェックリスト・写真撮影・メモ・周辺成約価格・ハザード）を1画面に。既存checklistJSON/内見写真機能の再パッケージ。
7. **売却出口シミュレーター** — `ssSimBest/Std/Worst` × `ssLoanBalance` を組み合わせ、「X年後に売ると残債±いくら」を1チャートに（データは取得済み、計算とUIのみ）。
8. **同建物の買い時ガイド** — `ssPriceJudgments`（住戸別判定）を「この建物で一番割安な住戸」カードに昇格。`duplicateCount`/`competingListingsCount` と組み合わせ「売り急ぎシグナル」を表示。
9. **家族コラボの強化** — 既存コメント機能に「夫婦それぞれの★評価」を追加し、比較画面に「ふたりの一致度」を表示。自分専用アプリの「家族」側の体験を作る。

### P3（将来・新規データ取得が必要）

10. **賃料相場と実質利回り** — 賃貸掲載のスクレイピングを追加し、「貸したら月いくら/表面利回り」を資産性タブに。賃貸vs購入シミュレーターの精度も向上。
11. **新耐震/旧耐震の自動判定と建替えリスク表示**（builtYearから判定可能、表示のみならP1相当）。
12. **再開発・都市計画情報** — 東京都都市計画GISとの突合（用途地域表示の発展形）。

### 削除・縮小の提案（「必要なものだけ」）

- ダッシュボードのスコア分布・エリアランキングの常時表示 → 週次カードに降格
- 初回ウォークスルー自動表示 → 廃止
- CSVエクスポート → マイリストのメニュー内に格納（ツールバーから撤去）
- 通知頻度0〜6回/日 → プリセット（朝 / 朝晩 / リアルタイムのみ）に簡素化
