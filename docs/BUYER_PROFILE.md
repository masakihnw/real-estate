# 買い手条件（AI 相談プロンプト用）

> **Single Source of Truth**: Supabase `buyer_profiles` テーブル
> 実運用の買い手条件（個人情報）は Supabase を直接参照すること。
> **このファイルにはプレースホルダのみを記載し、実データを書かない。**

AI に物件購入の相談をする際、プロンプトに自動反映される買い手条件。
アプリ内の「買い手条件を設定」から編集し、Supabase に同期する。

実行時は以下の優先順位で読み込まれる:

1. **Supabase `buyer_profiles` テーブル** — 正（Routine 2 / iOS アプリ / scraping-tool が参照）
2. **`scraping-tool/config/buyer_profile.json`** — Supabase 不通時のフォールバック（**プレースホルダのみ**）
3. **`BuyerProfile.swift` preset** — 新規インストール時のデフォルト（**プレースホルダのみ**）

## 項目一覧（スキーマ）

| 項目 | 内容（プレースホルダ） |
|---|---|
| 家族構成 | 夫（YYYY年生）・妻（YYYY年生）、子ども○人 |
| 子ども計画 | 人数・想定時期・教育方針 |
| 働き方・通勤 | 各人の勤務形態（勤務地は環境変数 `COMMUTE_OFFICES_JSON` / Supabase で管理） |
| 世帯年収 | ○○○○万円 |
| 自己資金 | ○○○万円（または「なし（フルローン）」） |
| 借入内容 | 借入方針（諸費用・物件価格・ペア借入の有無など） |
| 金利タイプ | 変動 / 固定 |
| 想定金利 | ○.○% |
| 返済期間 | ○○年 |
| 月額制約 | 月返済・月総額の上限や制約条件 |
| 重視する点 | 資産性・間取り・広さ・エリア等の優先順位 |
| 住み替え計画 | 住み替え理由・サイクル・売却/賃貸方針 |
| 購入時期 | 購入時期の目安 |
| リスク許容度 | リスク許容度の説明 |
