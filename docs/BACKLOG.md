# UI/UX 刷新 バックログ

`docs/UIUX_REDESIGN_PROPOSAL.md` の Phase 1〜5 実装後に**残った／意図的に見送った項目**の一元管理。
Phase 1〜5 の中核（タブ再編・今日タブ・AIブリーフ・保存フィルタ通知・スワイプ強化・ウィジェット・
詳細5タブ・お金統合・ソート刷新・比較再設計・内見モード・地図フル刷新）は実装・main マージ済み。

## 管理方法

- **単一ソース**: 本ファイル（`docs/BACKLOG.md`）。提案書 §4（機能提案 P1〜P3）・§5（削除/縮小）から派生。
- **運用**: 着手時に該当項目を「状態」列で `進行中` にし、完了で `✅ 済 (PR #xx)` を付記。
  粒度が大きい／期日管理したい項目は着手時に GitHub Issue へ昇格（本ファイルからリンク）。
- **優先度の目安**: P2（次フェーズ） > 縮小提案の未実施 > P3（新規データ取得が必要）。
  運用・手動確認はリリース前チェックとして別管理。
- **同期**: 提案書を改訂したら本ファイルも更新。完了項目は履歴として残す（消さない）。
