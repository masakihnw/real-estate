# 手動実行（1. 取得0件の原因特定）結果

## 実行内容

- `python3 main.py --source both -o /tmp/out.json`
- 続けて `--no-filter --max-pages 1` で SUUMO の生データを取得

## 結果

### 原因

**SUUMO はパースできているが、全件が「条件通過0件」で除外されている。**

- ログ: `SUUMO: ページNで19件パースしたが条件通過0件（価格・地域・間取り等で除外）。`
- フィルタなしで SUUMO 1ページ → **25件** 取得可能。

- 価格・専有・間取り・築年・徒歩も条件（7,500万〜1億、55〜75㎡、2LDK〜3LDK、1982年以降、10分以内）から外れている。

### 結論

1. **東京23区フィルタは意図どおり動作している**  
   住所に「〇〇区」が含まれない物件は正しく除外されている。

2. **現在の SUUMO 一覧URLは「東京都全体」**  
   `jj/bukken/ichiran/` の `ta=13` は東京都全体のため、多摩地域（青梅・八王子等）が多く、23区の物件が後ろのページに多い、または件数が少ない可能性がある。

3. **取得0件の意味**  
   - 現状の一覧では、**条件を満たす23区の物件がヒットしていない**、または  
   - 23区の物件が **100ページ安全上限より後ろ** にあり到達していない、のどちらか。

## 今後の候補

- **SUUMO の 23区限定URLを利用する**  
  `/ms/chuko/tokyo/city/` や 23区東部（`sa_other_02`）など、23区・地域別のURLがあれば、そこから一覧を取得するように変更すると、無駄なページ取得が減る。
- **連続して条件通過0件のときに打ち切り**  
  例: 5ページ連続で条件通過0件なら打ち切る（23区以外が続いていると判断）。  
  取得件数0のときの実行時間短縮と負荷軽減になる。
