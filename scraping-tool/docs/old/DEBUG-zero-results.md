# 取得件数0件のときの確認方法

取得件数が0件になる主な原因は次の2つです。

## 1. 一覧ページのHTML構造が変わった（パース0件）

SUUMO/HOME'S の一覧ページのクラス名やレイアウトが変わると、パース結果が0件になります。

**ログの例**
- `SUUMO: ページ1で0件パース。一覧のHTML構造が変わった可能性があります。`
- `HOME'S: ページ1で0件パース。一覧のHTML構造が変わった可能性があります。`

**確認方法**
- フィルタを外して実行し、生データが取れるか確認する。
  ```bash
  python main.py --source suumo --no-filter -o /tmp/suumo_raw.json
  python main.py --source homes --no-filter -o /tmp/homes_raw.json
  ```
- どちらも0件なら、該当サイトの一覧HTMLを開き、`property_unit-content` / `cassetteitem`（SUUMO）や `mod-mergeBuilding`（HOME'S）などのセレクタがまだ使えるか確認する。

## 2. 全件が条件で除外された（パースはできているが通過0件）

価格・専有・間取り・築年・徒歩・東京23区のいずれかの条件で、パースした物件がすべて除外されている場合です。

**ログの例**
- `SUUMO: ページ1でN件パースしたが条件通過0件（価格・地域・間取り等で除外）。`
- `HOME'S: ページ1でN件パースしたが条件通過0件（価格・地域・間取り等で除外）。`

**確認方法**
- 上記と同様に `--no-filter` で実行し、生データに物件が含まれているか確認する。
- `config.py` の条件（価格帯・専有面積・築年・徒歩・東京23区）を見直す。

## まとめ

| ログ | 原因 |
|------|------|
| 「0件パース」 | 一覧のHTML構造変更。サイトの一覧ページを確認し、セレクタ修正が必要。 |
| 「N件パースしたが条件通過0件」 | 条件が厳しすぎる、または住所/URLで23区以外が混ざっている。`--no-filter` で生データを確認し、条件を調整。 |
