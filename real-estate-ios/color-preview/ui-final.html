SwiftUI 実装プロンプト（全画面分）

以下の確定UIデザインを SwiftUI で実装してください。
アクセントカラー: #007AFF（iOS System Blue）
新築価格色: #34C759（System Green）
カード/セクション背景: Color(white: 0.973) ≈ #F8F8FB
ライトモード固定（.preferredColorScheme(.light)）

===================================
■ 一覧画面（ListingRowView / ListingListView.swift）
===================================
カードスタイル。白背景の上に薄グレーカード。アイコンなしテキストのみ。

- ForEach + List 内の各カード
- カード: VStack(alignment: .leading, spacing: 5)
  1行目: HStack { OwnershipCircle(所有権=青丸「所」/定借=オレンジ丸「借」, 18pt丸, 白文字) + 物件名 + 価格(accent.bold) + 騰落率バッジ/儲かるバッジ + Spacer + [掲載終了バッジ] + [📷枚数] + [💬件数] + ♥ }
  2行目: HStack { 間取り 面積 築年/入居 階/階建 戸数 } .font(.system(size:11)).foregroundStyle(.secondary).lineLimit(1)
  3行目: 路線名「駅名」徒歩X分（※路線名・駅名・徒歩分数のみ。住所等は不要） .font(.system(size:10)).foregroundStyle(.tertiary)
  4行目: ハザードバッジ（⚠アイコンなし、チップのみ）
  5行目: 通勤時間バッジ（[PGロゴ] xx分 / [M3ロゴ] xx分）— hasCommuteInfo のみ
- 掲載終了カード: opacity(0.75)
- .safeAreaInset(edge: .bottom) でカプセル型「更新中…」インジケーター（非ブロッキング）
- ツールバー右: 更新ボタンのみ。フィルタ・並び替えは右下オーバーレイ（地図画面と同様）

===================================
■ 詳細画面（ListingDetailView.swift）
===================================
グループセクション（Apple Settings 風）。セクション順序:
① 掲載終了バナー（isDelisted のみ）
② 物件名 + ♥ボタン
③ 住所（右端に Google Maps アイコンボタン。タップで物件名を Google Maps 検索。丸型ボタン: map.fill + accentColor 背景）
④ コメント（コメント一覧 + 入力フィールド。作成者名・アバター・相対時間表示。家族間共有）
⑤ 内見写真（PhotoSectionView: カメラロール選択 + カメラ撮影。横スクロールサムネイル。タップでフルスクリーン。✕で削除確認）
⑥ 物件情報（1列リスト: DetailRow 左ラベル/右値。間取り/面積、所在階/階建を統合）
  - 最寄駅（先頭、展開/折りたたみ式）
  - 価格, 間取り/面積, 築年/入居時期, 所在階/階建, 総戸数, 権利形態, 種別
  - 月額支払いシミュレーション（物件情報グリッドの下。金利0.8%/50年/頭金0で計算。中古のみ表示。LoanCalculator.monthlyPayment使用）
⑥-b 通勤時間（tintedGlassBackground: blue tint。hasCoordinate のみ。MKDirections transit で計算）
  - オフィスA: 所要時間・乗換回数・経路概要
  - オフィスB: 所要時間・乗換回数・経路概要
  - 平日朝8:00出発で計算、7日キャッシュ
⑦ 住まいサーフィン評価（tintedGlassBackground: blue tint）
  - 中古: 沖式中古時価 + 値上がり率 + レーダーチャート（本物件 青 + 行政区平均 グレー破線）
  - 新築: 儲かる確率 + 沖式新築時価 + 割安判定
  - レーダーチャート: ssRadarData JSON（偏差値ベース）。ない場合はフォールバック計算
  - ランキング: 「3位 / 12件中」形式
⑦-b シミュレーション（SimulationSectionView, tintedGlassBackground: blue tint。新築のみ表示。条件: 頭金0・金利0.8%・50年ローン前提。住まいサーフィンの値上がりシミュレーションは新築ページにのみ存在）
⑧ ハザード情報（tintedGlassBackground: orange tint。hasHazardRisk のみ）
⑨ 掲載終了物件 → 削除メッセージ / 通常 → 外部サイトボタン（Button + SFSafariViewController）

- SFSafariViewController でログイン状態を共有（Link ではなく Button + sheet）
- .padding(.horizontal, 14)

■ フィルタ画面（ListingFilterSheet.swift）
===================================
アコーディオン折りたたみ式。各セクションのヘッダーをタップで展開/閉じ。

【重要な変更】駅フィルタ → エリア（区）フィルタに変更:
- ListingFilter の stations: Set<String> を wards: Set<String> に変更
- フィルタロジック: listing.address から区名を抽出して比較

構造:
  .presentationDetents([.medium, .large])
  ScrollView {
    VStack(spacing: 0) {
      // 各フィルタセクション: DisclosureGroup スタイル
      FilterAccordion("価格帯", summary: priceSummary) {
        CustomRangeSlider(...)
      }
      FilterAccordion("間取り", summary: layoutSummary) {
        FlowLayout { ForEach(layouts) { FilterChip(...) } }
      }
      FilterAccordion("駅徒歩", summary: walkSummary) {
        Slider(...)
      }
      FilterAccordion("広さ", summary: areaSummary) {
        Slider(...)
      }
      FilterAccordion("権利形態", summary: ownershipSummary) {
        FlowLayout { FilterChip("所有権") FilterChip("定期借地") }
      }
      FilterAccordion("エリア（区）", summary: wardSummary) {
        // エリアごとにグループ化した 3列グリッド
        ForEach(wardGroups) { group in
          Text(group.area).font(.caption2).foregroundStyle(.tertiary)
          LazyVGrid(columns: 3) {
            ForEach(group.wards) { ward in
              WardToggleButton(ward, isSelected: ...)
            }
          }
        }
      }
    }
  }
  .safeAreaInset(edge: .bottom) {
    Button("\(count)件の物件を表示") { dismiss() }
  }

FilterAccordion コンポーネント:
  @State var isExpanded = false
  VStack(spacing: 0) {
    Button { withAnimation { isExpanded.toggle() } } label: {
      HStack {
        Text(title).font(.subheadline).fontWeight(.semibold)

■ 地図画面（MapTabView.swift）
===================================
既存実装を維持。変更点:
- フィルタの stations → wards 変更に追随
- ピンのフィルタリングも ward ベースに
- store.isRefreshing 中は画面下部にカプセル型「更新中…」インジケーター表示（地図操作はブロックしない）
- ポップアップ: 物件名の左に OwnershipCircle(18pt丸バッジ, 所有権=青「所」/定借=オレンジ「借」) を配置。サブテキスト行からは権利形態表記を削除

■ お気に入り画面
===================================
ListingListView(favoritesOnly: true) を使用。
- 掲載状態チップフィルタバー: すべて / 掲載中 / 掲載終了（各件数表示）
- カードフォーマットは一覧と同一
- 掲載終了カード: opacity(0.75) + オレンジ「掲載終了」バッジ
- 空状態: ContentUnavailableView("お気に入りがありません", systemImage: "heart.slash", description: "...")

===================================
■ 設定画面（SettingsView.swift）
===================================
Form ベースの5セクション構成。
セクション:
1. 通知: 通知頻度 Stepper（0〜6回、0=OFF）+ 各回の時刻 DatePicker + 未通知の新着件数表示 + コメント通知 Toggle（ON/OFF）。前回通知からの累積新着をまとめて通知。コメント通知は他ユーザーがコメントした際に即時通知し、タップで該当物件の詳細画面に遷移。全テキスト黒色（.foregroundStyleなし）
2. データ: @Query で中古/新築件数を動的表示 + 最終更新 + フルリフレッシュ（.alert 確認モーダル付き）
3. 詳細設定: ①スクレイピング条件（タップで ScrapingConfigView を .sheet 表示。価格・専有面積・築年・駅徒歩・総戸数・間取り（チップ選択）・路線（チップ選択、未選択=全路線対象）を Firestore scraping_config/default に保存。次回の GitHub Actions スクレイピングから反映）②DisclosureGroup でカスタム URL（通常非表示）+ info.circle ボタンで説明 alert
4. アカウント: Google アカウント名・メール + ログアウトボタン（確認 alert 付き）
5. このアプリについて: Bundle.main から version + build
- didBecomeActiveNotification で通知状態を自動再チェック
